Overview

This document outlines the product requirements for the Q2 sprint, focusing on the new user onboarding experience and key infrastructure improvements. Our goal is to reduce time-to-value for new signups from 12 minutes to under 3 minutes.

Key Requirements

  • • Implement guided onboarding flow with progress tracking
  • • Integrate Clerk for authentication (replacing custom auth)
  • • Build template gallery for first-run workflow creation
  • • Add contextual help tooltips throughout the app — linked to @Ship onboarding v2

Timeline

Phase 1 ships by April 15. Phase 2 (analytics + iteration) begins April 20. Full rollout expected by end of Q2.
