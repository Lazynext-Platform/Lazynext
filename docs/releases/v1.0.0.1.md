# v1.0.0.1 ‚Äî Hardening

Observability + E2E follow-up to [v1.0.0.0](https://github.com/Lazynext-Platform/Lazynext/releases/tag/v1.0.0.0). No product surface changes.

## Added

- **Structured observability on `scoreDecision`** ‚Äî every call emits a single-line JSON log with `event`, `source`, `provider`, `model_version`, `duration_ms`. On fallback, `fallback_cause` distinguishes `no_ai_keys` / `llm_call_failed` / `json_parse_failed` so you can alert on the exact shape of a regression.
- **E2E test suite for `/d/[slug]`** (`tests/e2e/public-decision.spec.ts`) ‚Äî 4 active tests covering the 404 path, the ISSUE-003 response-time ceiling, console-error cleanliness, and navigable 404 pages. 1 skipped test shows the shape to unskip once test Supabase creds are wired into CI.
- **Launch-surface smoke tests** (`tests/e2e/launch-surface.spec.ts`) ‚Äî locks in that `/blog`, `/blog/launching-lazynext`, and `/changelog` render the real v1.0.0.0 content, and that unknown blog slugs return a clean static 404.
- 3 new unit tests locking in the decision-scorer log shape so alerting rules stay valid.

## Changed

- `scoreDecision` internals: split the monolithic `try/catch` into distinct AI-call and JSON-parse blocks. Previously both failure modes landed in the same bare `catch {}` with no way to tell them apart in prod.
- `/blog/[slug]` now uses `dynamicParams = false` so unknown slugs 404 statically instead of falling through to dynamic SSR.

## Fixed

- VERSION file synced to `1.0.0.1` (the observability chore bumped `package.json` but left `VERSION` at `1.0.0.0`).

## Operator note

Pipe `decision_scorer` log lines to your aggregator (Axiom / Datadog / Vercel drains / etc.) and graph `source:heuristic` by `fallback_cause`. A spike in `json_parse_failed` means Llama is returning unparseable output again ‚Äî tune the prompt or extend `extractJson()`. A spike in `llm_call_failed` means Groq or Together is down ‚Äî the heuristic keeps the product running either way.
