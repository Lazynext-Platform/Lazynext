## The Problem With JavaScript Video Editors

JavaScript is great for UI. It's terrible for real-time video compositing.

Every JavaScript-based video editor we studied had the same bottleneck: the main thread. When you're compositing 17 blend modes, applying 6 GPU effects, rendering MSDF text, and computing optical flow — all at 60fps — a garbage-collected, single-threaded runtime hits a wall.

## Enter Rust + WebAssembly

Our compositor is a single Rust crate (`rust/crates/compositor`) that compiles to:

1. **WebAssembly** — Runs in the browser via `wasm-bindgen`, accessing WebGPU directly
2. **Native binary** — Runs on desktop with full `wgpu` access and DeckLink SDI output
3. **UniFFI library** — Called from Kotlin/Swift on mobile

The same 1,070-line compositor code powers all three platforms. No duplication. No platform-specific workarounds.

## What Rust Gives Us

### Zero-Cost Abstractions
Our CRDT merge algorithm processes 10,000 operations in under 1ms because Rust's ownership model eliminates GC pauses. There's no garbage collector to interrupt a real-time video pipeline.

### Fearless Concurrency
The GPU compositor, audio DSP engine, and CRDT sync all run on separate threads. Rust's type system guarantees no data races — at compile time. Our `Arc<Mutex<NLEState>>` pattern means the editor state is shared safely across a dozen subsystems.

### WASM Binary Size
The entire compositor — 17 blend modes, 6 effect shaders, MSDF text rendering, ACES color science, stereoscopic 3D — compiles to 17 MB of WASM. That's smaller than a single JPEG thumbnail in some apps.

## The Architecture

```
User Input → React UI → WASM Bridge → Rust CRDT Engine
                                          ↓
                                    GPU Compositor (wgpu/WebGPU)
                                          ↓
                                    Canvas / SDI Output
```

Every editing operation flows through the same path. The React UI is a thin shell — all business logic lives in Rust. This means:

- The desktop app shares 100% of editing logic with the web app
- AI agents (via MCP) use the same interface as human editors
- Testing the engine doesn't require a browser
