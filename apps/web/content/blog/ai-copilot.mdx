## Beyond "Generate a Video"

Most AI video tools stop at generation. Type a prompt, get a clip. That's useful, but it's not editing.

Chronos Copilot does what a human editor does: it understands your intent, decomposes it into concrete steps, executes each one against the right tool, and verifies the result. All from natural language.

## The Orchestrator

When you type "cut the silence from this interview, add cinematic color grading, and overlay b-roll of the city," Chronos doesn't just call one AI model. It:

1. **Decomposes** — Breaks your intent into discrete editing steps using an LLM (OpenAI, Anthropic, or local Ollama)
2. **Executes** — Calls the right microservice for each step: Whisper for transcription, SAM2 for rotoscoping, Replicate for b-roll generation
3. **Repairs** — If a step fails, it retries with adjusted parameters (our Agentic Repair Loop)
4. **Verifies** — Checks the CRDT operation log to confirm every edit was applied

## Powered by Rust, Accessible by Prompt

Every operation Chronos generates is a CRDT delta — the same deterministic, conflict-free data type that powers real-time collaboration. This means:

- AI edits are just another peer in the collaboration session
- Human editors can undo any AI operation instantly
- The full timeline state is inspectable and auditable

## The Tool Chain

Chronos has access to 40+ editing tools, from basic operations like "add track" and "trim clip" to advanced AI tasks:

| Tool | Backend |
|------|---------|
| Transcribe | OpenAI Whisper |
| Dub | ElevenLabs TTS |
| Rotoscope | SAM2 (CPU/ONNX) |
| B-Roll | Stable Video Diffusion / Pexels |
| Style Transfer | OpenCV / Diffusers |
| Stem Separation | Demucs → Spleeter → Librosa HPSS |
| Upscale | RealESRGAN → FFmpeg Lanczos |
| Color Grade | ACES 1.3 pipeline |
| Audio Enhance | Parametric EQ + Compression DSP |

And every tool degrades gracefully — no API key? It falls back to local processing.

## What's Next

We're expanding Chronos to support multi-turn conversations, project-level awareness ("remember I prefer teal-orange grading"), and direct-to-social publishing.

The goal isn't to replace editors. It's to make the first 80% of editing — the tedious stuff — instantaneous, so humans can focus on the creative 20%.
